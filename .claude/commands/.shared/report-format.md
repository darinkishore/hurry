# Standard Report Format

Use this structure for reporting results:

```markdown
# {Command Name} Results

## Summary
[Brief overview of what was done]

## Key Findings

### ✅ Success / Preserved / Passed
[List of things that are working correctly]

### ⚠️ Warnings / Concerns
[List of potential issues or areas needing attention]

### ❌ Issues / Failures
[List of specific problems that must be fixed]

## Verification Status
- Build: [pass/fail/skipped]
- Format: [pass/fail/skipped]
- Clippy: [pass/fail/skipped]
- Tests: [pass/fail/skipped/not run]

## Next Steps
[Clear recommendation: ready to proceed, needs fixes, needs manual review, or next command in workflow]
```

Adapt sections as needed for the specific command, but maintain this general structure for consistency.
